| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-01 | {Tampering} | CLI dag pause/resume | accept | dagID is provided as a positional argument. Invalid dagID returns ErrDAGNotFound from the store layer. No SQL injection or command injection risk — Go cobra handles arg parsing, and dagID is passed directly to the NATS KV store. |
| T-04-02 | {Information Disclosure} | CLI error messages | accept | Error messages include dagID and current DAG status (per existing API wrapper design). dagIDs are not secrets. DAG status is already visible via `dag ls`. No additional disclosure. |
| T-04-03 | {Denial of Service} | Unauthenticated NATS access | accept | ebctl connects to NATS without authentication by default. This is a dev/ops tool running on localhost or controlled networks. Production deployments should configure NATS auth separately (out of scope for this phase). |

  ```go
      // ----- pausing→paused auto-transition (SG-04) -----
      if state.Meta.Status == DAGStatusPausing && !state.HasInFlightSteps() {
          meta, rev, err := s.wf.Store.GetMeta(ctx, state.Meta.ID)
          if err != nil {
              return err
          }
          if meta.Status != DAGStatusPausing {
              return nil // concurrent Resume or Cancel changed it; benign (D-14)
          }
          meta.Status = DAGStatusPaused
          meta.PausedAt = time.Now().UTC()
          if err := s.wf.Store.PutMeta(ctx, state.Meta.ID, meta, rev); err != nil {
              if errors.Is(err, ErrStaleRevision) {
                  return nil // benign CAS race (Resume won) — sweep handles it
              }
              return err
          }
          // DAG is now paused — do NOT call maybeFinalize (D-13: paused is not terminal)
          return nil
      }
      // ----- end pausing→paused -----
  ```

  **Critical timing:** This check goes AFTER step transitions (MarkDone/MarkFailed/cascade) and `enqueueReady` (which checks the guard added in Task 1), but BEFORE `maybeFinalize`. This means:
  - The completed step is marked done in state ✓
  - Cascade-skipped steps are persisted to store ✓
  - No new dispatches happen (enqueueReady guard blocks) ✓
  - The DAG transitions to paused if no more in-flight steps ✓
  - maybeFinalize is NOT called for paused DAGs ✓

  **Change 2: Sweep recovery for pausing/paused DAGs (SG-03, per D-16/D-17/D-18)**

  Replace the simple filter at lines 100-113 (the `for _, dag := range dags` loop body) with a status switch:

  ```go
  for _, dag := range dags {
      switch dag.Status {
      case DAGStatusRunning:
          state, err := s.loadState(ctx, dag.ID)
          if err != nil {
              continue
          }
          s.mu.Lock()
          ready := state.ReadyToRun()
          err = s.enqueueReady(ctx, state, ready)
          s.mu.Unlock()
          _ = err

      case DAGStatusPausing:
          // D-16: Recovery for leader crash while pausing — transition to paused.
          state, err := s.loadState(ctx, dag.ID)
          if err != nil {
              continue
          }
          if state.HasInFlightSteps() {
              continue // still draining; do nothing
          }
          meta, rev, err := s.wf.Store.GetMeta(ctx, dag.ID)
          if err != nil {
              continue
          }
          if meta.Status != DAGStatusPausing {
              continue // concurrent writer changed it
          }
          meta.Status = DAGStatusPaused
          meta.PausedAt = time.Now().UTC()
          _ = s.wf.Store.PutMeta(ctx, dag.ID, meta, rev) // CAS; benign fail on race

      case DAGStatusPaused:
          // D-18: Skip non-terminal paused DAGs — zero CPU.
          // D-17: Auto-finalize if all steps are terminal.
          state, err := s.loadState(ctx, dag.ID)
          if err != nil {
              continue
          }
          terminalStatus, done := state.Terminal()
          if !done {
              continue // not all terminal — skip (zero CPU per D-18)
          }
          meta, rev, err := s.wf.Store.GetMeta(ctx, dag.ID)
          if err != nil {
              continue
          }
          if meta.Status != DAGStatusPaused {
              continue
          }
          meta.Status = terminalStatus
          _ = s.wf.Store.PutMeta(ctx, dag.ID, meta, rev)

      default:
          continue // done/failed/canceled — existing behavior
      }
  }
  ```

  Note: The auto-finalize in sweep (D-17 override of D-04) is necessary because a paused DAG where all steps became terminal during the drain would otherwise stay paused forever with no way to progress.

  **Import check:** Add `import ("time")` if not already present (needed for `time.Now().UTC()` in auto-transition).

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Scheduler ↔ StateStore | Internal CAS-mediated boundary — scheduler reads/writes DAGMeta and StepRecords via StateStore (MemStore or NatsStore). Both are co-located in-process. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-01 | Spoofing | handleEvent guard | accept | No authentication boundary — DAG status is read from in-process StateStore. All writers share the same process context. |
| T-02-02 | Tampering | CAS transition (pausing→paused) | mitigate | CAS with revision check prevents stale overwrites. If concurrent Resume changes status to running, the auto-transition CAS fails with ErrStaleRevision and returns nil (benign). Sweep also uses CAS for the same reason. |
| T-02-03 | Repudiation | Event ACK during pause | accept | No event audit trail needed at this layer. Step outcomes are durably stored in KV by StepHook before the event is published. |
| T-02-04 | Information Disclosure | Sweep auto-finalization | accept | No sensitive data in DAG status transitions. Status values are lowercase string constants with no payload. |
| T-02-05 | Denial of Service | Event flood during pause | mitigate | Guard at handleEvent entry returns nil for all events on pausing/paused DAGs — events are consumed (ACKed) without processing. No CPU wasted on state transitions. |
| T-02-06 | Elevation of Privilege | Sweep status mutation | accept | Sweep runs in the same process with the same privileges as the scheduler. CAS prevents conflicting writes. |
</threat_model>

  **Test 1: `TestScheduler_Pause_BlocksDispatch`** (SG-01, SG-02)

  Verify that after pausing a running DAG, step completion events are consumed but no new steps are enqueued.

  ```
  1. Create DAG with chain: a -> b (3-step: a, b, c where a depends on nothing, b depends on a, c depends on b)
  2. Submit DAG, wait for root step (a) to be enqueued
  3. Manually set meta to DAGStatusPausing
  4. Complete step a via emulateHook
  5. Assert: no new enqueues (still 1), step b is still StatusPending in store
  6. Complete step a again (idempotent) — assert no change
  ```

  **Test 2: `TestScheduler_Pause_TransitionsToPaused`** (SG-04)

  Verify that when the last in-flight step completes while pausing, meta transitions to paused.

  ```
  1. Create DAG with single step: a
  2. Submit DAG, wait for enqueue
  3. Manually set meta to DAGStatusPausing
  4. Complete step a via emulateHook
  5. Poll meta until status == DAGStatusPaused (use polling pattern from TestScheduler_Finalizes_DAG_Status)
  ```

  **Test 3: `TestScheduler_Pause_InFlightContinues`** (SG-01)

  Verify that a running step completes normally during pause (result is persisted).

  ```
  1. Create DAG with single step: a
  2. Submit DAG, wait for enqueue
  3. Manually set meta to DAGStatusPausing
  4. Complete step a via emulateHook with result []byte("42")
  5. Verify: step a is StatusDone in store
  6. Verify: result for step a is []byte("42") in store via wf.Store.GetResult
  ```

  **Test 4: `TestScheduler_Pause_StepAddedDuringPause`** (SG-02)

  Verify that a dynamic step added during pause stays pending (not enqueued) until resume.

  ```
  1. Create DAG with single step: a
  2. Submit DAG, wait for enqueue
  3. Manually set meta to DAGStatusPausing
  4. Add step b via store.PutStep (simulating dynamic step)
  5. Publish step-added event for step b
  6. Assert: no enqueues for step b (still 1 total)
  7. Verify: step b is StatusPending in store
  ```

  **Test 5: `TestScheduler_Sweep_HandlesPausingDAG`** (SG-03)

  Verify that sweep transitions a pausing DAG (with no in-flight steps) to paused.

  ```
  1. Seed store manually with:
     - Meta: DAGStatusPausing
     - Step a: StatusDone
  2. Create Workflow with toggleElector (leader=false), captureEnq
  3. Start scheduler
  4. Set elector to leader=true
  5. Poll: meta transitions to DAGStatusPaused
  6. Assert: 0 enqueues (no ready steps to enqueue — a is already done)
  ```

  **Test 6: `TestScheduler_Sweep_SkipsPausedDAG`** (SG-03)

  Verify that sweep skips paused DAGs with non-terminal steps (no enqueues).

  ```
  1. Seed store manually with:
     - Meta: DAGStatusPaused
     - Step a: StatusPending (ready to run — deps satisfied since no deps)
  2. Create Workflow with toggleElector (leader=false), captureEnq
  3. Start scheduler
  4. Set elector to leader=true
  5. Wait and assert: 0 enqueues (paused DAG, skip per D-18)
  ```

  **Test 7: `TestScheduler_Sweep_AutoFinalizesPausedDAG`** (SG-03)

  ```
  3. Start scheduler
  4. Set elector to leader=true
  5. Poll: meta transitions to DAGStatusDone
  6. Assert: 0 enqueues
  ```
